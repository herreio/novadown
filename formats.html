MARC 21

005 - Date and Time of Latest Transaction

  • LOC: Bibliographic (Full / Concise)

Field Definition and Scope

Sixteen characters that indicate the date and time of the latest record transaction and serve as a version identifier for the record. They are recorded according to Representation of Dates and Times (ISO 8601). The date requires 8 numeric characters in the pattern yyyymmdd. The time requires 8 numeric characters in the pattern hhmmss.f, expressed in terms of the 24-hour (00-23) clock.

…

The date on which a record is first entered into machine-readable form is contained in field 008/00-05. The Date entered on file never changes.

Input Covention

System Generation – Field 005 should be system generated.

008 - Fixed-Length Data Elements-General Information

  • LOC: Bibliographic (Full / Concise)

Field Definition and Scope

Forty character positions (00-39) that provide coded information about the record as a whole and about special bibliographic aspects of the item being cataloged. These coded data elements are potentially useful for retrieval and data management purposes.

…

Data elements are positionally-defined. Character positions that are not defined contain a blank (#). All defined character positions must contain a defined code; for some field 008 positions, this may be the fill character (|). The fill character may be used (in certain character positions) when a cataloging organization makes no attempt to code the character position. The fill character is not allowed in field 008 positions 00-05 (Date entered on file). …

Character positions 00-17 and 35-39 are defined the same across all types of material …

00-05 - Date entered on file

  • LOC: Bibliographic (Full / Concise)
Guidelines for Applying Content Designators

Computer-generated, six-character numeric string that indicates the date the MARC record was created. Recorded in the pattern yymmdd.

… The date entered on file in 008/00-05 is never changed. The date and time of latest transaction information in field 005 changes each time a transaction is made to the record. The latest transaction information enables an organization handling more than one version of a record to identify the most current version. The fill character (|) is not allowed in any of these positions. Field 008/00-05 is usually system generated.

Input Convention

Capitalization – Alphabetic codes are input in lowercase.

Field length – Field 008 should always consist of forty (40) character positions.

7110 / 209A/$x00-09 - Signatur

  • K10plus Format-Dokumentation

Signatur

Pica3 Pica+ wiederholbar Marc 21
$a $a Nein 852 $c / 924 $g

Bei Bestellkatalogisaten kann anstelle der Signatur ein Kommentar (z.B. bestellt) erfasst werden.

Ausleihindikator (nur SWB)

Pica3 Pica+ wiederholbar Marc 21
$D $D Nein 852 $m / 938 $k / 924 $d

Bei der Belegung von $D wird unterschieden, ob ein Exemplarsatz mit der Kennung l in E001 oder mit der Kennung e in E001 belegt wird. In Exemplarsätzen mit der Kennung l können in $D nur die unten beschriebenen festgelegten Codes vergeben werden. Bei Exemplarsätzen mit der Kennung e kann freier Text eingegeben werden.

Für die Online-Fernleihe werden nur die mit l gekennzeichneten Exemplarsätze berücksichtigt.

Folgende Codes für eine Ausleihbeschränkung sind definiert:

Indikator Bedeutung
e Erwerbungsdaten
l Nur für den Lesesaal
p Präsenzbestand
n Nicht verleihbar
s Für die Benutzung gesperrt
u Sonstige Ausleihbeschränkung
v Nicht verfügbar

E001 / 208@ - Neuanlagedatum und Selektionsschlüssel

  • K10plus Format-Dokumentation

Das Feld E001-E999 ist das erste (Pflicht-)Feld eines jeden Exemplardatensatzes (E001 für den 1. Exemplarsatz, E002 für den 2. Exemplarsatz usw. bis E999 für den 999. und letzten Exemplarsatz) und enthält das Erfassungsdatum und den Selektionsschlüssel.

Eingegeben wird nur der Selektionsschlüssel. Das Erfassungsdatum des Exemplarsatzes wird automatisch vom System ergänzt.

Neuanlagedatum

Pica3 Pica+ wiederholbar Marc 21
ohne $a Nein –

Das Unterfeld wird vom System besetzt und enthält das Datum in der Form TT-MM-JJ.

Indexierung
Indextyp (IKT) Indexschlüssel (IKT) Routine
SEL (8880) SEL (8880) Datum
SXN (4015) SXN (4015) Phrase

Selektionsschlüssel

Pica3 Pica+ wiederholbar Marc 21
_:_ $b Nein –
Besonderheiten für SWB-Bibliotheken

Zugelassener Inhalt für Pos. 1 = l oder e.
Pos. 2-3 ist numerisch, erlaubte Inhalte 01-99; Pos. 2-3 bei Pos. 1 = e muss identisch sein mit einem Pica-Exemplarsatz mit Pos.1 = l unter der gleichen ELN.

Bei Neuanlegen eines Satzes muss es manuell mit dem Inhalt Status belegt werden. Das Neuanlagedatum wird maschinell bei Speicherung des Satzes vergeben.

Status = l für Lokalsatz auf Pos. 1
Pos. 2-3 laufenden Nummerierung des Lokalsatzes, linksbündig und zweistellig.
Status = e für Exemplarsatz auf Pos. 1
Pos. 2-3 linksbündig und zweistellig die Nummerierung des Lokalsatzes.
Status = z für aus der ZDB übernommenen Bestandssatz (wird bei Übernahme maschinell gesetzt, ergänzt durch die zweistellige Nummer der Sortierhilfe aus ZDB-Feld 8032)
Status = zez für aus der ZDB übernommenen Bestandssatz, der aus der EZB über den gemeinsamen Lieferdienst „EZB ZDB“ übertragen wurde.
